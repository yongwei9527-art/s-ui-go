# DNS 防泄露模式说明与操作指南

本文档说明面板中 DNS 防泄露功能的设计目标、模式差异、配置步骤和检查项含义，适用于需要评估或调整 DNS 出口策略的部署场景。

## 1. 功能定位

DNS 防泄露功能用于降低应用、系统或客户端绕过 sing-box DNS 模块直接发起 DNS 查询的风险。启用后，面板会在配置归一化、模板应用和状态检查过程中协助维护以下配置：

- 配置加密远程 DNS，默认标签为 `remote-dns`。
- 保留本地 DNS，默认标签为 `local-dns`，用于 `.lan`、`.local` 等本地域名。
- 在路由规则中补齐 DNS 协议劫持和 TCP/UDP 53 端口劫持。
- 检查 DNS 服务器、DNS 规则、路由规则和严格模式解析器配置是否满足预期。

该功能主要保护核心服务端配置。JSON 订阅可以携带 DNS 与路由规则；`ss://`、`trojan://`、`vless://` 等单链接通常无法表达完整 DNS 与路由保护策略。

## 2. 模式说明

| 模式 | 适用场景 | 配置行为 |
| --- | --- | --- |
| 关闭 | 需要完全手动维护 DNS 与路由规则 | 不自动补齐 DNS 防泄露配置；检查功能仍可用于查看当前状态。 |
| 推荐 | 通用部署场景 | 使用加密远程 DNS、DNS 劫持和本地域名规则；尽量保留已有可用解析器配置。 |
| 严格 | 对 DNS 出口一致性要求较高的场景 | 在推荐模式基础上，强制 `route.default_domain_resolver` 使用 `remote-dns`。 |

推荐模式侧重兼容性，严格模式侧重默认域名解析出口的一致性。启用严格模式前，应确认局域网域名、多网卡环境、网关或透明代理场景不会依赖其他默认解析器。

## 3. 调整为严格模式

1. 进入面板的 **DNS 防泄露** 页面。
2. 在顶部 **防泄露模式** 下拉框中选择 **严格**。
3. 等待下拉框保存状态结束。模式选择会自动写入设置项 `dnsLeakGuardMode=strict`。
4. 刷新页面，确认下拉框仍显示 **严格**。
5. 如需同步重写页面中的 DNS 服务器、DNS 规则和路由 DNS 劫持规则，点击 **应用模板**。
6. 应用模板后，点击页面下方 **保存**，将 DNS 与路由配置写入数据库。
7. 点击 **防泄露检查** 右侧刷新按钮，确认检查状态。

模式本身会自动保存；模板应用只修改当前配置内容，仍需点击 **保存** 后才会写入 DNS 与路由配置。

## 4. 刷新后模式回退的原因与处理

早期交互中，防泄露模式依赖页面下方 **保存** 按钮提交。如果只切换下拉框后直接刷新，前端状态不会写入数据库，因此会回到默认的推荐模式。

当前逻辑已调整为模式切换后自动保存。若仍出现刷新后回退，可按以下顺序排查：

1. 确认切换后等待保存状态结束再刷新。
2. 在浏览器开发者工具中确认 `api/save` 请求返回成功。
3. 检查设置表中 `dnsLeakGuardMode` 的值是否为 `strict`。
4. 如旧数据库缺少 `dnsLeakGuardMode` 设置项，后端保存逻辑会在提交时自动创建该设置项。

## 5. 应用模板的影响范围

点击 **应用模板** 会在当前页面配置中写入标准 DNS 防泄露模板，主要影响以下内容：

| 配置区域 | 调整内容 |
| --- | --- |
| DNS 服务器 | 写入 `remote-dns` 和 `local-dns`。 |
| DNS 规则 | 将 `.lan`、`.local` 等本地域名路由至 `local-dns`。 |
| DNS 默认出口 | 设置 `dns.final = remote-dns`。 |
| DNS 基础选项 | 设置推荐的缓存、过期、独立缓存和反向映射选项。 |
| 路由规则 | 补齐 DNS 协议劫持和 TCP/UDP 53 端口劫持规则。 |
| 严格模式 | 设置 `route.default_domain_resolver = remote-dns`。 |

模板会替换当前页面中的 DNS 服务器和 DNS 规则。已有自定义 DNS 规则时，建议先备份配置，再决定是否应用模板。

## 6. 推荐操作流程

仅切换严格模式：

1. 选择 **严格**。
2. 等待自动保存完成。
3. 刷新页面确认模式保持不变。

同步应用标准防泄露模板：

1. 选择 **严格**。
2. 点击 **应用模板**。
3. 确认模板覆盖提示。
4. 点击页面下方 **保存**。
5. 刷新检查结果，确认状态满足预期。

## 7. 检查项说明

| 检查项 | 含义 |
| --- | --- |
| 远程加密 DNS | 是否存在 `remote-dns`，且类型为 TLS、HTTPS、QUIC 或 H3 等加密 DNS。 |
| 本地 DNS | 是否存在 `local-dns`，用于处理本地域名解析。 |
| 最终 DNS | `dns.final` 是否指向 `remote-dns`。 |
| 本地域名规则 | `.lan`、`.local` 是否被路由至 `local-dns`。 |
| DNS 协议劫持 | DNS 协议流量是否由 `hijack-dns` 接管。 |
| 53 端口劫持 | TCP/UDP 53 端口流量是否由 `hijack-dns` 接管。 |
| 自动检测出口网卡 | `route.auto_detect_interface` 是否启用。 |
| 严格模式域名解析器 | 严格模式下 `route.default_domain_resolver` 是否为 `remote-dns`。 |

## 8. 注意事项

- DNS 防泄露检查反映当前服务端配置状态，不等同于客户端系统级 DNS 泄露测试。
- 浏览器内置 DoH、操作系统安全 DNS、路由器 DHCP 下发 DNS 等外部因素可能影响最终解析路径。
- Tun、TProxy、透明代理和多网卡环境需要结合实际路由策略验证。
- 生产环境调整前建议导出配置备份，并在客户端侧进行 DNS 泄露测试。
